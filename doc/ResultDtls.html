WAY4 UFX ResultDtls Specification

In requests, the ResultDtls element is used to specify the additional data that should be sent in the response. The following table contains the subset of ResultDtls codes.

#CodeTypeDescriptionRequest
Common Codes
1DateFromDate (yyyy-MM-dd)Start date of a time interval.Statement
2DateToDate (yyyy-MM-dd)End date of a time interval.Statement
Information & Application Response Services Codes
3BalanceString If the assigned value is 'Y' in a request, the Balance object is returned in the response. It may be used if it is supported by the operation. If additional types balance needed, then code of these types should be presented here separated by ',' mark. Contract Inquiry, Doc
Financial Service Codes
4StmtContTypeString Parameter for statement inquiry for filtering operations by their status. Possible values:
  • "Posted" – posted operations,
  • "Auth" - authorization operations,
  • "All" – all operations,
  • "LastOperations" – the same as "All". But the count of operations is limited to "LastOperationsNumber" and default sort order is descending
See also: LastOperationsNumber. 		Statement
5LastOperationsNumberNumber Count of operations to be returned in response.
Maximum value is 10. 		Statement
6StmtTypeString The assigned value specifies type of statement that is returned in the response. Supported values:
  • "Mini" – mini statement,
  • "Additional" – additional statement,
  • "Custom" – custom statement.
